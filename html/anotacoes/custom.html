Custom

  • Para criar uma anotação devemos usar designação @interface.

  • O nome de uma anotação deve começar com letra maiúscula, podendo ser ser composto por mais do que um nome. Cada nome pode começar com letra maiúscula.

  • A anotação terá métodos, no entanto esses métodos não terão qualquer implementação. Os métodos não devem ter a cláusula throws.

  • Podem ser definidos valores por omissão associados a cada método. O mesmo é dizer que é opcional e quando são definidos representam-se pela palvra chave default.

  • Os parâmetros não podem ser null.

  • O tipo de retorno dos métodos dve ser primitivo, enum, string, outra qualquer classe ou array, neste último o array pode assumir tipos primitivos, num, string ou classe.
