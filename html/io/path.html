Path

  • A classe Path representa ficheiros e diretorias como objetos imutáveis.

  • Disponibiliza o método of como construtor de objetos Path.

  • Objetos Path podem representar caminhos absolutos ou relativos.

  • Podemos criar objetos Path a partir de caminhos inexistentes, contudo poderemos ter problemas a partir do momento que usamos esses objetos.

  • Outros:

    • Root podem representar-se por / ou C:, por exemplo. Depende do sistema operativo.
    • Cada nós do caminho pode ser seprado por / ou \, dependendo do sistema operativo.
    • Caminho relativos começam na pasta definida.
    • O símbolo . representa a diretoria atual.
    • O símbolo .. representa a diretoria pai.
