Sistema de ficheiros

  • O package java.nio.file contém um conjunto de classes que nos permitem interagir com o sistema de ficheiros dos dispositivos.

    • Path - representa ficheiros e diretorias
    • Files - disponibiliza operações para atuar sobre objetos Path
    • FileSystem - permite obter informação sobre os ficheiros do sistema

  • Por esse motivo serialização não é uma boa solução para armazenar informação no longo termo.

  • A falta de correspondência entre a definição da classe no procesos de serialização pode ter um comportamento imprevisível.

  • o objeto da classe ObjectInputStream analise a variável serialVersionUID. Esta constante da classe, que deve ser definido quando a classe implemetna a interface Serializable, é usado para controlo de versão da classe. No momento de leitura se não existir correspondência é lançada a exceção InvalidClassException e o processo é interrompido.

  • A classe java.io.File é legacy.
