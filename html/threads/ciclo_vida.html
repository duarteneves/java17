Ciclo de vida

  • É possível ir de qualquer estado da thread para runnable exceto terminated.

  • A partir de runnable é possível ir para qualquer estado exceto new.

  • Descrição dos ciclos de vida

    • new - thread instânciada
    • runnable - thread em execução
    • blocked - thread bloqueada até a monitorização dar um sinal que já não existe bloqueio
    • waiting - thread em espera até receber sinal de outra thread em execução
    • time_waiting - thread em espera durante um determinado tempo
    • terminated - thread terminada

  • As alterações de estados não são imediatas, o processamento da thread precisa de alocar uma slot no CPU disponível.

  • A mesma instância de runnable pode ser lançada por mais do que uma thread.

  • É possível verificar se uma thread ainda não terminou.

  • A mesma thread não pode arrancar mais do que uma vez.
