Conceitos

  • Uma thread é um caminho de execução.

  • Cada thread é um objeto que impleta a interface Runnable através de implementação do método run.

  • Uma thread é lançada através da invocação do método start.

  • As threads permitem a execução concorrente e particionada pelos diferentes CPUs existentes.

  • Não é garantido que uma thread, que seja lançada em primeiro lugar, termine antes de uma thread lançada posteriormente.

  • Se o teu programa for single thread não irá tirar partido dos múltiplos CPUs.
