Propriedades

  • As threadas podem ser definidas por um nome personalizado e cada thread tem um identificador único.

  • Uma thread pode ser categorizada como um deamon ou user (default).

  • Daemon threads são de baixa prioridade e por esse motivo são executadas em background. As user threads foram desenhadas para executarem tarefas complexas, enquanto que as daemon threads devem ser usadas para suporte ou monitorização de outras tarefas.

  • Enquanto existirem user threads a JVM não termina, no entanto, não existindo este tipo de threads, mesmo que existam daemon threads a JVM termina e força estas threads a terminar.

  • Pode ser definida uma prioridade a cada thread. A prioridade não garante a ordem de execução, mas determina o número de slots de CPU que lhe vai ser dedicado.

    • Thread.MAX_PRIORITY (10)
    • Thread.MIN_PRIORITY (1)
    • Thread.NORM_PRIORITY (5)(default)

  • O método setDeamon só pode ser invocado antes da thread avançar para runnable.

  • O método join espera a thread corrente até a thread que invoca o método terminar.
