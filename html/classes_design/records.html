Records

  • Records surgem a partir do Java 14.

  • Objetivos e vantagens: diminuir escrita de código, diminuir o uso de bibliotecas como Lombok, garantir imutabilidade, usar records para serializar / desserializar.

  • Um record é uma class final, por esse motivo não pode ser uma subclasse.

  • Um record implicitamente estende a classe java.lang.Record .

  • Todos os atributos são private e final, consequentemente são imutáveis .

  • Um único construtor é criado com todos os atributos definidos no record.

  • Um record automaticamente disponibiliza métodos para acesso aos atributos. Contudo, não usam a terminologia get, apenas identificam o nome do atributo.

  • Um record automaticamente disponibiliza métodos hasCode(), equals() e toString().

  • Usar records para serializar / desserializar, como data-provider de uma base de dados, em mensagens enviadas e recebidas na comunicação assíncrona.
