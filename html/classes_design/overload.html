Overload

  • Dois ou mais métodos com o mesmo nome e o mesmo tipo de retorno na mesma classe.

  • Métodos com um número diferente de argumentos e argumentos cujos tipos diferem.

  • O nome dos argumentos é irrelevante.

  • Overload permite reduzir a complexidade da API de uma classe uma vez que não é necssário redefinir o nome dos métodos quando a única diferença é o número ou tipo de argumentos.
