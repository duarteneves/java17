Controlo de fluxo

if / else

  • O bloco de código dentro do if é executado se a expressão boolean definida devolver true.

  • Por outro lado, o else é executado caso a expressão boolean do if devolver false.

  • A definição do else é opcional.

  • Não existe else / if em Java, nós podemos é usar um if / else dentro de outro if / else.

operador condicional ternário

  • Permite fazer afetações condicionais substituindo if / else.

switch (antes Java 17)

  • Quando estamos perante vários cenários de control, mas esses cenários têm por base a mesma variável, em vez de criarmos múltimos if/else podemos usar o switch que cóigo mais fácil de ler e manter.

  • switch aceita expressões do seguinte tipo: byte, short, int, char, String e enum. Até à versão 7 do Java apenas aceitava int e a partir da vesão 8, int, strings e enums.

  • O switch identifca os casos do valor esperado presente na expressão do switch, cada case é uma constante. A execução segue o fluxo a que a expressão do switch correspondente tendo em conta a label do case.

  • A execução continua até encontrar a palavra break.

  • Caso não exista correspondência entre a expressão e um dos casos considerados, o flow caí no caso de default definido. A definição de default é opcional mas é aconcelhável, nem que seja para lançamento de uma exceção, caso não esperado.
