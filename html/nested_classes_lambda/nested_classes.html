Classes Nested

  • Em Java é possível definir classes dentro de classes. A estas classes chamamos de nested classes.

  • Existem as static e non-static nested classes, estas últimas designam-se por inner classes.

  • Dependendo do contexto em que são usadas, existem 4 tipos de nested classes:

    • static (nested) - associada ao contexto estático da outer class.
    • member (inner) - associada dentro do contexto da instância da outer class.
    • local (inner) - associada ao contexto de um método específico.
    • anonymous (inner) - implementação inline ou estenção de uma interface ou classe.

  • Classes nested do tipo static e member podem ser definidas como private, public, protected ou default.

  • A utilização destas classes não é comum, mas existem 3 razões que justificam a sua utilização:

    • organização: por vezes a definição de uma classe só faz sentido no contexto de outra classe, especialmente quando a nested class não é usada fora do contexto da outer class.
    • acesso: as nested classes têm acesso especial a parâmetros das outer classes.
    • conveniência: não é necessário criar um novo ficheiro java para a nested class quando esta é usada apenas no contexto da outer class.
