default, static e private

  • Como já vimos anteriormente, as interfaces podem imlementar métodos "não abstratos".

  • As expressões lambda apenas consideram os métodos abstratos disponibilizados por interfaces funcionais.

  • Métodos default e static definidos pela interface são válidos na definição da API, mas não na expressão lambda.

  • Métodos private podem estar presentes, mas não são visíveis fora da interface.

  • Não há a necessidade de sobrepor a implementar de métodos default a não ser que entrem em conflito com a implementação de outras interfaces.

  • As expressões lambda não causam conflito o conflito anterior, uma vez que cada implementasção é inline e sobre uma única interface.
