Logging

  • A Java API de logging permite identificar cada logger a partir de um nome.

  • É prática comum usar-se o nome da classe como nome do logger.

  • Existem 7 nívels de logging que permitem ao programador "logar" informação de acordo com a categoria, importância ou classificação do cenário ou mensagem.

  • Os métodos de logging da API podem ser agruados em 5 categorias:

    • log - Certamente o método mais comum e usado. Permite a definição de mensagens de logging através da identificação da categoria, da própria mensagem e, de forma opcional, de parâmetros presentes na mensagem.
    • logp - Método similar ao anterior, mas que permite, adicionalmente, especificar os nome da class e do método.
    • logrb - Método similar ao anterior, mas que permite, adicionalmente, especificar resource bundle para localizar a mensagem de logging.
    • entering, exiting, throwing - Métodos usados para marcar a entrada de um método, a saída do método e o lançamento de exeções. A categoria de logging por trás desta defiição é FINER.
    • severe, warning, config, info, fine, finer, finest - Métodos que integram a categoria que o programador pretende aplicar.
