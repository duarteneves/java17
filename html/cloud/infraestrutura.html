Infraestrutura

  • As insfraestruturas mais antigas eram definindas por um servidor, Linux ou Windows, uma JVM, um servidor aplicacional como Tomcat, JBoss ou Weblogic, e por último, o artefacto da nossa aplicação Java.

  • Hoje em dia as novas infraestruturas em cloud, privadas ou públicas têm definidos os seguintes componentes:

    • servidores - máquinas reais ou virtuais com o respetivo sistema operativo
    • docker - containers que permitem a virtulização de ambientes
    • kubernetes - gestor do deployment da aplicação que permite escalar a solução e controlar os containers docker
    • JVM - permite o ambiente para execução de aplicações Java
    • application server - servidor aplicacional
    • aplicação - código fonte de aplicação Java
