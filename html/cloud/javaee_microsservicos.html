Java EE vs Microsserviços

  • Por um lado temos as aplicações Java EE a correr sobre Weblogic que se definem pelas seguintes caraterísticas:

    • todas as operações correm sobre o mesmo servidor aplicacional
    • disponibiliza containers Web e EJB
    • disponibiliza módulo de segurança, concorrência, gestor transacional, etc...

  • Do outro lado temos mais uma das últimas tendências que é a implementação de microsserviços. Neste âmbito o Java recomenda a integração da biblioteca Helidon, sendo que, podem existir 2 tipos de aplicação, Java SE e Java MP Server.

    Java SE:

    • servidor com capacidade limitadas
    • hosting de aplicações microsserviços

    Java MP Server:

    • integra um container "leve"
    • disponibiliza um sub conjunto de features Java EE
    • hosting de aplicações microsserviços
