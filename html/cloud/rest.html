Rest

  • Um serviço Rest representa um ou mais recursos da aplicação.

  • Cada recurso representa uma entidade de negócio e deve ser mapeado com o seu url.

  • Cada recurso deve ter definido um conjunto de operações mapeada nos métodos HTTP: GET, POST, PUT e DELETE.

    • GET - obter um conjunto de elementos ou o detalhe de um elemento específico
    • POST - cria um novo elemento
    • PUT - atualiza o elemento identificado
    • DELETE - remove o elemento identificado

  • As resposta Rest podem ser representadas em diferentes formatos, nomeadamente JSON, Texto ou XML.
