JDBC (Java Database Connectivity)

  • A API JDBC disponibiliza uma interface para interagir com base de dados. Esta interface está definida no package java.sql.

  • Não é específica de um provider como MySQL, Oracle, SQL Server, Derby, H2 ou qualquer outro.

  • O driver JDBC é uma implementação da interface disponível na API.

  • A configuração do acesso a uma base de dados implica a integração do driver JDBC de acordo com o provider e versão da base de dados para melhor integração. O driver deve estar disponível na class path ou module path da aplicação.

  • O driver deve ser selecionado no momento de criação da base de dados e é apresentado sobre o formato de url:

  • A API JDBC disponibilza as seguintes API:

    • DriverManager - classe para gerir o driver e obter a conexão
    • Connection - interface representativa da sessão com a base de dados
    • Statement - interface que representa a instrução SQL
    • PreparedStatement - interface que representa a instrução SQL pré compilada
    • CallableStatement - interface que representa invocações de funções ou procedimentos SQL
    • ResultSet - interface representativa do conjunto de linhas, resultado da execução da instrução SQL

  • Todas as interfaces implementação AutoCloseable.

  • Operações JDBC podem lançar a exceção SQLException.
