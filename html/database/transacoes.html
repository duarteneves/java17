Transações

  • Transações definem uma série de ações sobre a base de dados. Se uma transação falhar, todas as outras ações sobre a mesma transação devem falhar, deixando o estado do programa inalterado. Correndo bem ou não a transação deve sempre terminar com a invocação de um dos seguintes métodos:

    • commit - todas as alterações são persistidas alterando o estado
    • rollback - descartas as alterações pendentes e liberta a connection

  • Por omissão as connections JDBC são auto-commit. O método commit é invocado sempre que em cada invocação de uma instrução SQL.

  • É possível definir save points através de invocação do método setSavePoint, neste caso o rollback caso seja invocado apenas descartas operações efetuadas depois do save point.
