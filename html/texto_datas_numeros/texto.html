Classes String e StringBuilder

Inicialização de strings

  • Representa uma sequência de caracteres. Não é um tipo primitivo, é como outro objeto Java.

  • Tem um modo único de ser inicializada, sem o new. Mais, não devemos mesmo fazer new String().

  • O Java otimiza a gestão de memória de strings.

Operações sobre strings

  • Objetos String são imutáveis, ou seja, são inalteráveis.

  • A API da classe String disponibiliza com um conjunto de métodos que retornam sempre uma nova String, mas não alteram a String original.

  • Por uma questão de conviniência podemos uar o operador + em vez do método concat, mas cuidado, o símbolo + é também um operador aritmético.

Indexação de strings

  • Sendo uma String uma sequência de caracteres, esses caracteres podem ser acedidos individualmente. O 1º caracter encontra-se no índice 0 da sequência e o último caracter pode ser encontrado no último índice que é igual ao comprimento da sequência - 1.

  • Os métodos substring(), indexOf(), lastIndexOf() e charAt() permitem-nos trabalhar uma string tendo por base a identificação dos índices da sequência.

  • O método substring, que recebe 2 argumentos, dois índices, o inicial e o final, não os considera da mesma forma. O índice inicial é inclusivo, enquanto que o índice final é exclusivo.

  • Os métodos indexOf ou lastIndexOf retornam -1 quando o caracter identificado não existe na string.

  • Quando o índice identificado não é válido, ou seja, é inferior a 0 ou superior ao tamanhao da sequência, é lançada uma exceção, StringIndexOutOfBoundsException.

StringBuilder

  • Os objetos da classe StringBuilder são mutáveis.

  • Disponibiliza os métodos substring, indexOf e charAt, mas também os métodos append, insert, delete e reverse que aceitam uma string como argumento.

  • Cenário de múltiplas concatenações, usar StringBuilder em vez de String.
