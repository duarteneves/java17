Processamento em paralelo

  • Os elementos da stream são dividios em subsets. Subsets podem ser dividos em mais subsets.

  • Os subsets são processados em paralelo.

  • A ordem de processamento é indeterminada.

  • Para ativar o paralelismo usar parallel(). Por omissão é sequencial (stream()).

  • A stream será processada de modo sequencial ou paralelo dependendo do último método que é invocado.

  • Para termos processamento paralelo o nosso código deve ser definido de forma a:

    • stateless - o estado de um elemento não deve afetar outro elemento
    • non-interfering - o código não deve ser afetado
    • associative - o resultado não deve ser afetado pela ordem das operações

  • Não é possível prever a ordem de processamento e a atribuição dinâmica de CPUs.
