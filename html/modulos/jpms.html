JPMS (Java Platform Module System)

  • A partir da versão 9, toda a API foi reorganizada em módulos.

  • O nome do módulo deve ser único e tipicamente é definido pelo nome do package onde se encontra definido.

  • Os packages dentro do módulo estão escondidos por omissão.

  • Dependências entre módulos são verificadas no arranque da aplicação. Erros de deploy como a falta da identificação de módulos são verificados no momento do arranque da aplicação.

  • Não é possível criar dependências circulares.

  • As classes de aplicações modulares são carregadas pela definição do module-path e não do class-path.
