XML

  • E como não há duas sempre três, é possível também a injeção de XML.

  • Não é tão comum como as anteriores, mas é igualmente possível através da definição dos ficheiros XML que contêm entidades ou namespaces.

  • Desta forma é possível:

    • efetuar referências recursivas provocando ciclos infinitos
    • adicionar url maliciosos
    • integrar outros ficheiros XML maliciosos

  • Para previnir estes ataques deve-se:

    • parar o processo de parse de XML quando são detados tags e valores inesperados
    • usar bibliotecas open source (OWASP) que permite sanitizar os dados XML
