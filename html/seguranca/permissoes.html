Permissões

  • A API Java permite a verificação de permissões sobre a nossa aplicação e respetivos recursos. Todas as permissões definidas no ficheiro java.policy descendem da classe java.security.Permission.

  • A recomendação é que se verifiquem permissões antes de atuar sobre os recursos da aplicação.

  • Se o acesso a determinado recurso for negado pela definição da política de segurança, a API lança uma AccessControlException.

  • Uma vez verificada a permissão a ação sobre o recurso deve fazer-se da seguinte forma:

    • definir o método estático doProvileged da classe AccessController
    • criar uma instância de PrivilegedAction e implementar o método run.

  • O bloco de código do método run deve conter apenas código que foi definido nas no ficheiro java.policy.

  • Fora deste bloco de código, não é suposto interagir com o recurso definido nas permissões.
