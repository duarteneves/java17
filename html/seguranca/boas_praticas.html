Boas práticas

  • Usar JPMS para proteger as classes de acesso via reflection.

  • Aplicar o princípio de encapsulamento de forma mais restrita possível.

  • Usar objetos imutáveis e aplicar a keyword final sempre que possível.

  • Operar sobre objetos clonados mantém objetos imutáveis.

  • Na definição de métodos de uma super classe defini-los para herança, caso seja esse o propósito, caso contrário marcar os métodos como final ou private.

  • Não invocar métodos overridable a partir de construtores.
