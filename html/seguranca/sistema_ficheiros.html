Sistema de ficheiros

Directory traversal attack

  • O uso paths relativos no acesso ao sistema de ficheiros pode ser perigoso, pois permite aceder a diretorias e ficheiros fora da diretoria raíz da aplicação.

  • Em vez de caminhos relativos, devemos usar os canonical path. Para evitar o uso de caminhos relativos podemos usar os os seguintes métodos da classe java.nio.Path:

    • normalize
    • toRealPath (criação de symbolic links)

DoS

  • Tipicamente este género de ataques carateriza-se por provocar aà aplicação processos demorados e recorrentes, partindo deste princípio são aconcelháveis as seguintes ações:

    • monitorizar o nº de operações de input / output sobre o sistema de ficheiros
    • verificar a capacidade do disco no servidor

  • Para prevenir cenários anteriores a equipa de desenvolvimento deve:

    • terminar operações que passem para lá de uma determinada quantidade de dados processada
    • definir um timeout para realizar a operação e assim libertar o recurso
